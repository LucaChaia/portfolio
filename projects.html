Projects

These are a few of my favorite technical projects. They're projects where I've built something challenging and fun. I've had to learn new skills both as a computer scientist and a professional to complete these projects.

Mancala Engine Project

What it is: A Python-based Mancala engine built for an artificial intelligence course, designed to test search strategies and decision-making algorithms in a structured game environment.

Tech stack: Python, Jupyter Notebook, NumPy, dataclasses, matplotlib, time, random.

Why I built it: I wanted to better understand how heuristic search works in practice, especially how an agent can evaluate future game states and choose moves strategically under constraints.

What I learned: This project taught me how much performance matters in search problems. Comparing plain minimax with alpha-beta pruning made it much clearer how the right optimization can dramatically reduce the number of states that need to be explored.

Heartland's Adventure

What it is: A short action-adventure game inspired by classic top-down games, featuring exploration, simple combat, light puzzles, and a dance button.

Tech stack: Godot 4, with art adapted from the Ninja Adventure asset pack.

Why I built it: I wanted to prove to myself that I could take a game idea from concept to completion. I'm heavily inspired by the current indie developer scene, and I wanted to contribute to it in whatever way I could.

What I learned: This project gave me a practical introduction to how games are built frame by frame, but it also taught me a lot about iteration. Feedback from other people made the game better, even when it forced me to revise ideas I was originally attached to. As Stephen King writes: "Kill your darlings, even when it breaks your egocentric little scribbler's heart."

Boulder County Nature Association Project

What it is: A nonprofit web application that included a wildlife search tool and an admin system for managing records. I worked primarily on the backend.

Tech stack: Flask, SQLite, and API-based backend development.

Why I built it: This project came through Blueprint Boulder, where our team develops software for a nonprofit partner. I was excited by the chance to work on something that would actually be useful to a real organization.

What I learned: I came into the project with little experience in Flask and SQLite, so it pushed me to learn quickly. It also helped me grow as a collaborator: using Git in a team setting, asking better questions, and contributing even when I was still figuring things out. I learned to strike a better balance between asking for help and learning things on my own.
